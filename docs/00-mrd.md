---

## 1. 核心问题（一句话）

**解决什么问题？**

AI 工程学习者缺乏完整的、可实战的项目案例来系统学习 AI Agent 开发——现有教程要么过于碎片化，要么只讲理论不涉及工程实践，导致学习者无法掌握构建完整 AI 系统的能力。

## 2. 目标用户（具体画像）

### 人群特征

| 用户类型 | 描述 | 比例 |
|----------|------|------|
| **求职转行者** | 有 Python 基础，希望进入 AI/大模型相关岗位，需要深度项目提升竞争力 | 40% |
| **后端工程师** | 从传统开发转向 AI 工程方向，需要补齐 Agent 与 RAG 能力 | 30% |
| **独立开发者** | 希望基于 AI 技术构建产品，需要完整的技术模板和架构参考 | 20% |
| **AI 产品经理** | 理解 AI 产品构建方式，了解 AI 能力如何转化为功能 | 10% |

### 痛点场景

1. **理论零散**：通过碎片化教程学习，缺乏系统性的知识体系
2. **缺乏实战**：只有理论课程和简单 Demo，没有可运行的完整项目
3. **架构盲区**：不知道如何设计一个完整的 AI 系统，各模块如何协同
4. **工程经验不足**：缺乏前端后端联调、流式输出、异步任务处理等工程实践经验
5. **技术栈混乱**：AI 技术更新快，不知道该学哪些技术、如何选择

### 当前替代方案

| 方案 | 优点 | 缺点 |
|------|------|------|
| **LangChain 官方教程** | 权威、全面、免费 | 缺乏完整项目案例，教程零散 |
| **付费 AI 课程** | 系统性强、有讲师指导 | 价格高（¥500-2000），项目简单 |
| **开源 Agent 项目** | 代码完整、可运行 | 文档不完善，学习曲线陡峭 |
| **技术博客/文章** | 轻量级、易获取 | 深度不够，缺乏实战细节 |

## 3. 解决方案（核心功能）

**MVP 三个核心功能：**

### ① 多 Agent 对话系统

- **支持 7 种 Agent 类型**：General Agent、React Agent、Plan-Execute Agent、MCP Agent、Skill Agent、Text2SQL Agent、CodeAct Agent
- **可视化对话流程**：实时展示 Agent 推理决策路径
- **流式输出体验**：SSE 实时推送，类似 ChatGPT 的交互体验
- **多轮对话管理**：完整的上下文处理和历史记录

**价值**：让学习者理解不同 Agent 的应用场景和实现原理

### ② RAG 知识库系统

- **全格式文档解析**：支持 PDF、Docx、Markdown、Txt 等多种格式
- **智能分块策略**：基于物理布局感知的智能分块，确保语义完整性
- **混合检索引擎**：向量检索 + 关键词检索 + Rerank 重排序
- **可视化知识管理**：文档上传、解析、向量化的全流程可视化

**价值**：掌握企业级 RAG 系统的完整实现

### ③ 工具与 MCP 扩展能力

- **内置工具集**：搜索、计算、时间等常用工具
- **自定义工具上传**：支持通过 OpenAPI/Swagger 上传自定义工具
- **MCP 服务器集成**：原生支持 Model Context Protocol，可动态挂载自定义 MCP 服务
- **Skill 机制**：Prompt 渐进式加载，按需教导模型处理特定业务逻辑

**价值**：理解 AI 系统的扩展能力边界和集成方法

## 4. 竞品情况

| 竞品 | 优势 | 劣势 | 差异化机会 |
|------|------|------|-----------|
| **LangChain 官方教程** | 权威性强、技术最新、完全免费 | 缺乏完整项目案例、教程零散、无工程实践指导 | 提供**完整可运行的系统**，涵盖前后端、数据库、部署等全栈内容 |
| **其他开源 Agent 项目**（如 AgentGPT、AutoGPT） | 功能丰富、社区活跃 | 文档不完善、学习曲线陡峭、代码质量参差不齐 | **强调教学属性**，提供详细的项目分析文档和代码注释 |
| **付费 AI 课程平台**（如极客时间、拉勾教育） | 系统性强、有讲师答疑、有证书 | 价格高（¥500-2000）、项目简单、技术栈过时 | **免费开源**，项目更接近真实工程环境 |
| **AI 垂直媒体/社区**（如量子位、机器之心） | 行业资讯、技术趋势、案例分享 | 缺乏实战内容、无法动手实践 | **实战导向**，提供可立即运行的项目代码 |

## 附录：项目技术概览

### 技术架构总览

```
用户层: Web 界面 / 对话系统
    ↓
对话层: 上下文管理 / 历史记录 / 流式输出
    ↓
Agent决策层: 任务拆解 / 执行规划 / 多Agent协作
    ↓
能力层: RAG检索 / Tools工具调用 / MCP扩展
    ↓
模型层: 大语言模型 / Embedding模型
    ↓
数据层: MySQL / Redis / 向量数据库
```

### 核心技术栈

**后端：** FastAPI + LangChain + SQLModel + Redis + MySQL

**前端：** Vue 3 + TypeScript + Element Plus + Pinia

**AI/ML：** LangChain、通义千问、OpenAI、Anthropic Claude

**存储：** ChromaDB/Milvus（向量）、MinIO/OSS（对象）
