## 5. 代码规范

### 5.1 通用规范

- **文件命名**：
  - Python: `snake_case.py`
  - Vue/TS: `PascalCase.vue` / `PascalCase.ts`
  - 配置文件: `kebab-case.*`

- **目录结构**：
  - 按功能模块划分目录
  - 相关文件放在同一目录下
  - 公共代码放在共享目录

- **代码风格**：
  - 使用格式化工具统一风格（Prettier/Black）
  - 使用代码检查工具保证质量（ESLint/Ruff）
  - 编写清晰的注释和文档字符串

### 5.2 前端代码规范

- **命名约定**：
  - 组件：`PascalCase` (如 `UserList.vue`)
  - 变量/函数：`camelCase` (如 `userName`)
  - 常量：`UPPER_SNAKE_CASE` (如 `API_BASE_URL`)
  - 文件：组件`PascalCase.vue`，工具`camelCase.ts`

- **目录约定**：
  - 组件目录包含主文件和样式文件
  - 页面目录包含主文件、私有组件和样式文件
  - 公共组件放在 `components/` 目录
  - 页面组件放在 `pages/` 目录

- **代码组织**：
  ```vue
  <template>
    <!-- 模板 -->
  </template>

  <script setup lang="ts">
  // 导入
  // 定义
  // 方法
  // 生命周期
  </script>

  <style scoped lang="scss">
  // 样式
  </style>
  ```

### 5.3 后端代码规范

- **命名约定**：
  - 类：`PascalCase` (如 `UserService`)
  - 函数/变量：`snake_case` (如 `get_user`)
  - 常量：`UPPER_SNAKE_CASE` (如 `MAX_RETRIES`)
  - 文件：`snake_case.py`

- **目录约定**：
  - 按功能模块组织代码
  - 每个模块包含 `__init__.py`
  - 相关功能放在同一目录下

```python
from agentchat.core.agents import ReactAgent
from agentchat.core.models import get_llm

# 创建Agent
llm = get_llm("conversation_model")
agent = ReactAgent(
    model=llm,
    system_prompt="你是一个有用的AI助手",
    tools=[search_tool, calculator_tool]
)

# 执行Agent
response = await agent.run("今天天气怎么样？")
```

### 9.2 RAG系统

**位置**: `src/backend/agentchat/services/rag/`

**核心文件**:
- `retrieval.py`: 检索服务
- `parser.py`: 文档解析
- `chunking.py`: 分块策略
- `handler.py`: RAG处理器

**使用示例**:
```python
from agentchat.services.rag.handler import RAGHandler

# 上传文档
await RAGHandler.upload_document(

# 检索
docs = await RAGHandler.retrieve(
    query="什么是AI？",
    knowledge_ids=["kb_xxx"],
    top_k=5
)
```
